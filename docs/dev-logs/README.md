# 个人开发日志

本目录用于存放每位成员自己的开发日志。

每个人都需要维护一个独立日志文件，例如：

```text
docs/dev-logs/zhangsan.md
docs/dev-logs/lisi.md
docs/dev-logs/wangwu.md
```

## 使用规则

- 每次开发前，先阅读本目录下所有成员的最近日志。
- 每次开发前，先同步 `main`，再创建或切换自己的任务分支。
- 每次开发结束后，更新自己的开发日志。
- 日志采用追加方式，不覆盖历史记录。
- 如果修改了接口、数据库字段、路由、公共组件或协作流程，必须在日志中明确说明。
- 如果还有未完成事项，也要写在日志里，方便其他成员和 AI 接上。

## 开发前检查清单

```text
同步 main
阅读自己的开发日志
阅读其他成员开发日志
确认当前任务没有重复开发
确认没有接口或数据库字段冲突
创建或切换任务分支
开始开发
```
