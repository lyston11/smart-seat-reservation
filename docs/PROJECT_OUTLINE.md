# 项目开发大纲

## 1. 项目定位

本项目是学院公共区域学习座位预约系统，面向整个学校使用。核心目标是在高峰期预约场景下保证座位状态准确、预约不超卖、资源可管理、使用过程可追踪。

## 2. 核心角色

- 学生：查看座位、预约座位、扫码签到、签退释放座位、查看自己的预约记录。
- 管理员：维护区域、座位、开放时段、预约规则，查看实时占用情况，释放异常占用座位。
- 系统：处理预约冲突、状态流转、超时释放、缓存同步和审计记录。

## 3. 技术路线

```text
前端：React + TypeScript + Vite + Ant Design
后端：Java 21 + Spring Boot 4 + MyBatis-Plus
数据库：MySQL 8.4
缓存：Redis 7
数据库迁移：Flyway
接口文档：springdoc-openapi
本地依赖：Docker Compose
测试：JUnit 5 + Vitest
CI：GitHub Actions
```

## 4. 开发阶段

- 初始化前端和后端目录。
- 配置 MySQL、Redis 的 Docker Compose。
- 建立数据库迁移脚本。
- 建立统一响应、异常处理和基础健康检查。
- 建立开发、部署和协作文档。

### 阶段二：座位资源管理

- 座位管理。
- 开放时段管理。
- 座位时段库存生成。

### 阶段三：预约主流程

- 学生查看座位状态。

- 使用数据库原子更新防止重复抢座。

当前骨架已提供创建预约、签到、签退、取消预约和手动释放超时预约接口。后续需要补定时任务、权限校验和管理员页面。

## 5. 高并发设计原则

- 座位最终状态以 MySQL 为准，Redis 只做缓存和限流。
- 预约成功必须依赖数据库原子更新，而不是先查再改。
- 核心预约 SQL 使用条件更新：

```sql
UPDATE seat_slots
SET status = 'RESERVED'
WHERE id = ? AND status = 'AVAILABLE';
```

- 影响行数为 1 表示预约成功，影响行数为 0 表示座位已被占用。
- 所有状态流转必须可追踪，关键表保留创建和更新时间。
- 后续可按楼栋、区域、日期拆分缓存 key，降低高峰期读取压力。
