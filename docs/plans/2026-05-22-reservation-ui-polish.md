# 预约端 UI 连贯性优化实施记录

## 目标

- 每张桌子的座位在学生座位图中按桌内顺序展示为 `1号`、`2号`、`3号`、`4号`，避免历史全局座位标签造成跳号。
- 在学生选座页补充楼层、区域、预约时段、桌座的连续选择路径，减少楼层地图和具体座位选择之间的割裂感。
- 保持手机端同一套响应式页面，不新增独立移动端页面。

## 范围

- 只调整预约端前端展示和测试。
- 不修改后端签到验证、二维码 token、IP 校验、预约状态机和数据库结构。
- 不修改其他成员开发日志。

## 实现

- 新增 `seatDisplay` 前端工具，集中处理桌内座位编号、桌号和选择路径文案。
- `SeatMap` 改为根据同一桌内的座位排序计算显示编号，保留无桌子旧数据的兜底显示。
- `SeatSlotsPage` 增加选择路径条，串联楼层、区域、预约时段和桌座。
- 样式增加响应式路径条，桌面多列展示，手机端单列堆叠。

## 扩展计划：桌椅方向与缩放

### 目标

- 让同一区域中横向长桌、纵向侧桌、旋转桌都能在学生座位图中清晰呈现。
- 为坐标布局增加缩放控件，支持放大、缩小和适配画布，缩放过程有平滑动画。
- 保持后端数据结构不变，继续使用 `tableWidthPx`、`tableHeightPx`、`tableRotationDeg`、坐标字段表达桌子形态。

### 实施步骤

1. 先在 `SeatMap.test.tsx` 增加失败测试，覆盖坐标布局出现缩放控件、默认比例、点击放大/缩小/适配后的画布样式。
2. 增加纵向桌和旋转桌测试，确认桌子根据宽高比例和旋转角度带有方向类名与 `--table-rotation` 样式变量。
3. 在 `SeatMap.tsx` 中为坐标布局增加局部缩放状态，并只在有坐标布局时展示控制条。
4. 将坐标画布包一层 viewport/content 结构，通过 `transform: scale(...)`、`transform-origin: top left` 实现缩放动画。
5. 更新 `main.css` 中座位图相关样式，补充横向/纵向桌视觉差异、缩放动画和手机端紧凑控制。
6. 更新 AmorLX 开发日志，运行 `npm run test -- SeatMap.test.tsx`、`npm run test`、`npm run lint`、`npm run build` 和浏览器预览。

## 扩展计划：管理员学生视角座位图

### 目标

- 管理员在桌子管理页能看到与学生端一致的桌椅和座位渲染，快速定位学生反馈的具体桌子与座位。
- 管理员视图只做定位和查看，不触发预约，不修改签到验证流程。
- 复用 `SeatMap` 的桌椅布局、桌内编号、横向/纵向/旋转桌和缩放控件，避免学生端和管理员端产生两套视觉规则。

### 实施步骤

1. 先在 `App.test.tsx` 中增加失败测试，打开 `/admin/tables` 后应出现“学生视角座位图”，并显示 `T01 · 1号`、座位启用/停用状态。
2. 扩展 `SeatMap` 支持管理视图文案和点击回调，避免管理员端按钮使用“预约”语义。
3. 在 `AdminTablesPage.tsx` 中将 `Seat[]` 适配成 `SeatSlot[]` 形态，使用 `ACTIVE -> AVAILABLE`、`INACTIVE -> UNPUBLISHED` 显示，保持桌内编号和布局一致。
4. 在管理员桌子页新增一块“学生视角座位图”，点击座位后在旁边显示所属桌子、座位编号、系统座位号和状态。
5. 更新样式、开发日志，并运行 `npm run test -- App.test.tsx`、`npm run test`、`npm run lint`、`npm run build` 和浏览器预览。

## 扩展计划：管理员建桌生成真实座位

### 目标

- 管理员新增桌子时，不只生成桌子平面预览，还要生成可预约的真实座位。
- 桌型预设支持 1、2、3、4 人桌，自定义桌型可以手动填写座位数。
- 对历史空桌保持非破坏性修复：只有启用且没有真实座位的桌子才会自动补座位，已有座位、预约和签到状态不被覆盖。

### 实施步骤

1. 在 `App.test.tsx` 增加失败测试，确认管理员新增桌子时请求体携带 `seatCount`。
2. 在 `StudyTableServiceTest` 增加失败测试，确认后端收到 `seatCount=4` 后会生成 `NORTH/NORTH/SOUTH/SOUTH` 的真实座位。
3. 扩展 `CreateStudyTableRequest`、`UpdateStudyTableRequest` 和前端 `tables.ts` 的请求类型，加入 `seatCount`。
4. 在 `StudyTableService` 中注入 `SeatMapper`，新增空桌自动建座位逻辑，并限制座位数范围为 `1-12`。
5. 在管理员桌子页增加 1 人桌预设和座位数字段，创建、编辑、保存布局时都保持桌型和座位数一致。
6. 更新接口文档、开发日志，并运行后端、前端测试与浏览器预览。
